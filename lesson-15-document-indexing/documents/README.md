# Документы для индексации

Эта папка содержит документы, которые будут проиндексированы системой базы знаний.

## Структура документов

1. **01-mcp-server-creation.md** — Создание MCP сервера на Kotlin
2. **02-embeddings-and-vector-search.md** — Эмбеддинги и векторный поиск
3. **03-document-chunking.md** — Разбивка документов на чанки
4. **04-knowledge-base-integration.md** — Интеграция базы знаний с LLM агентом
5. **05-best-practices.md** — Лучшие практики работы с базой знаний

## Добавление новых документов

Просто поместите Markdown файлы в эту папку и запустите индексацию через API:

```bash
curl -X POST http://localhost:8080/api/indexing/index-directory \
  -H "Content-Type: application/json" \
  -d '{"directoryPath": "documents"}'
```

## Формат документов

Документы должны быть в формате Markdown (`.md`). Рекомендуется:

- Использовать заголовки для структурирования
- Разбивать текст на параграфы
- Использовать списки и код-блоки где уместно
- Добавлять метаданные (если нужно)

## Индексация

После индексации документы будут:

1. Разбиты на чанки (по 800 токенов с перекрытием 100 токенов)
2. Преобразованы в эмбеддинги через Ollama
3. Сохранены в базу знаний (SQLite)
4. Доступны для семантического поиска
