# ai_challenge_november

Хранилище реализаций ДЗ для AI Challenge по изучению работы с ИИ агентами и построению агентных систем.

## Структура репозитория

Каждое задание хранится в отдельной папке по следующему принципу:

```
lesson-XX-brief-description/
├── README.md          # Описание задания, требования, план реализации
├── .env.example       # Пример конфигурации окружения (скопируйте в .env)
├── .env               # Конфигурация окружения (не в git, создайте из .env.example)
├── config/            # Папка с примерами конфигурации (опционально)
├── server/            # Серверная часть (Kotlin + Ktor или Python)
├── client/            # Клиентская часть (Web, Telegram бот, или мобильное приложение)
└── docs/              # Дополнительная документация (опционально)
```

### Нумерация уроков
- Уроки нумеруются последовательно: `lesson-01-`, `lesson-02-`, и т.д.
- После номера идет краткое описание задания на английском (kebab-case)

### Языки программирования

**Основной язык:** Kotlin (Ktor) - для серверной части (привычен)

**Альтернатива:** Python - для сложных проектов с ML/NLP библиотеками (есть опыт небольшой)

### Конфигурация окружения

Каждый урок использует `.env` файлы для хранения конфигурации:
- **`.env.example`** - пример конфигурации (хранится в репозитории)

**Настройка:**
1. Скопируйте `.env.example` в `.env` в корне урока
2. Заполните `.env` файл своими значениями (API ключи, URL, модель и т.д.)
3. Конфигурация загружается автоматически при запуске приложения

Поддержка dotenv реализована через библиотеку `dotenv-kotlin` для Kotlin проектов.

### Структура каждого задания

1. **README.md в папке задания** - содержит:
   - Описание задания
   - Требования
   - План реализации
   - Инструкции по запуску

2. **server/** - серверная часть:
   - HTTP API для взаимодействия с агентами
   - Интеграция с AI провайдерами
   - Логика работы агентов

3. **client/** - клиентская часть (1 или 2 варинта):
   - Web интерфейс (HTML/CSS/JS)
   - Telegram бот
   - Мобильное приложение (опционально)

## Шаблон проекта

Для быстрого старта используйте готовый шаблон: **[lesson-00-project-template](./lesson-00-project-template/)**

Шаблон содержит:
- Базовую структуру проекта (Clean Architecture)
- Настроенный Ktor сервер (CORS, JSON, логирование)
- Конфигурацию для загрузки `.env` и YAML файлов
- Контроллер для отдачи статических файлов клиента
- Структуру папок для всех слоев архитектуры
- Gradle конфигурацию с необходимыми зависимостями

**Использование:**
```bash
# Скопируйте шаблон
cp -r lesson-00-project-template lesson-XX-description

# Настройте конфигурацию
cd lesson-XX-description
cp .env.example .env
# Заполните .env файл
```

Подробнее: [lesson-00-project-template/README.md](./lesson-00-project-template/README.md)

## Как работать с заданиями

1. Скопировать шаблон проекта: `cp -r lesson-00-project-template lesson-XX-description/`
2. Заполнить README.md с описанием задания
3. Настроить конфигурацию (`.env`, `config/`)
4. Реализовать согласно построенному плану
5. Документировать результаты в README задания
