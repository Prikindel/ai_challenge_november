# Примеры аналитических вопросов

Примеры вопросов для разных типов данных и рекомендации по их формулировке.

## Вопросы для логов

### Частота ошибок

**Вопросы:**
- "Какая ошибка встречается чаще всего?"
- "Какие топ-5 ошибок по частоте?"
- "Сколько раз встречается ошибка 'Connection timeout'?"
- "Какая ошибка самая критичная?"

**Ожидаемый ответ:**
```
Согласно анализу логов, наиболее часто встречающаяся ошибка: 
'Connection timeout' (45 случаев, 30% от общего числа ошибок).

Топ-5 ошибок:
1. Connection timeout - 45 случаев (30%)
2. Database error - 30 случаев (20%)
3. Authentication failed - 25 случаев (17%)
4. Network error - 15 случаев (10%)
5. Timeout exceeded - 10 случаев (7%)
```

## Рекомендации по формулировке вопросов

### ✅ Хорошие вопросы

**Конкретные:**
- "Какая ошибка встречается чаще всего?"
- "Сколько пользователей в городе New York?"
- "Какой день недели самый активный?"

**Структурированные:**
- "Какие топ-5 проблем по частоте?"
- "Сравни текущую неделю с предыдущей"
- "Сгруппируй данные по категориям"

**С метриками:**
- "Какая средняя длительность операций?"
- "Какой процент успешных запросов?"
- "Сколько записей за последний день?"

### ❌ Плохие вопросы

**Слишком общие:**
- "Проанализируй данные" (неконкретно)
- "Что здесь?" (непонятно)
- "Покажи всё" (слишком широко)

**Неконкретные:**
- "Что не так?" (неясно, что искать)
- "Есть ли проблемы?" (слишком общий вопрос)
- "Как дела?" (не относится к данным)

**Слишком сложные:**
- "Сделай полный анализ всех данных и найди все проблемы" (слишком широко)
- "Предскажи будущее" (не относится к анализу данных)

## Советы

1. **Будьте конкретны** — чем конкретнее вопрос, тем лучше ответ
2. **Используйте метрики** — спрашивайте про числа, проценты, сравнения
3. **Задавайте структурированные вопросы** — "топ-5", "сравни", "сгруппируй"
4. **Избегайте общих вопросов** — "проанализируй" слишком широко
5. **Используйте примеры** — смотрите на примеры выше для вдохновения
