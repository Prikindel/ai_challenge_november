# Основы RAG (Retrieval-Augmented Generation)

## Что такое RAG?

RAG (Retrieval-Augmented Generation) - это техника, которая объединяет поиск информации с генерацией текста. Вместо того чтобы полагаться только на знания модели, RAG сначала ищет релевантную информацию в базе знаний, а затем использует её для генерации ответа.

## Как работает RAG?

1. **Индексация**: Документы разбиваются на чанки и для каждого чанка генерируется эмбеддинг (векторное представление).

2. **Поиск**: Когда пользователь задает вопрос, генерируется эмбеддинг для запроса и находится наиболее похожие чанки по косинусному сходству.

3. **Генерация**: Найденные чанки используются как контекст для LLM, который генерирует ответ на основе этой информации.

## Преимущества RAG

- Актуальность: информация всегда актуальна, так как берется из базы знаний
- Точность: ответы основаны на реальных документах
- Прозрачность: можно видеть источники информации
- Масштабируемость: легко добавлять новые документы

## Использование в God Agent

God Agent использует RAG для поиска в персональной базе знаний. Документы организованы по категориям:
- Проекты
- Обучение
- Личное
- Справочники

## Лучшие практики

1. Разбивайте документы на логические чанки (500-1000 символов)
2. Используйте перекрытие между чанками (100-200 символов)
3. Добавляйте метаданные к чанкам (категория, источник, дата)
4. Регулярно обновляйте индексы при изменении документов
