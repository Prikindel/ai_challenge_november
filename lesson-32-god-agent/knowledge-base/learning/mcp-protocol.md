# MCP (Model Context Protocol)

## Что такое MCP?

MCP (Model Context Protocol) - это протокол для взаимодействия между AI агентами и внешними сервисами. Он позволяет AI агентам использовать инструменты и получать доступ к данным из различных источников.

## Архитектура MCP

MCP состоит из двух основных компонентов:
- **MCP Server**: предоставляет инструменты и ресурсы
- **MCP Client**: использует инструменты и ресурсы

## Типы MCP серверов

### 1. Git MCP
Работа с git репозиториями: чтение файлов, получение информации о ветках, коммитах.

### 2. Telegram MCP
Отправка сообщений в Telegram, создание напоминаний.

### 3. Analytics MCP
Анализ данных из различных источников: CSV, JSON, базы данных.

### 4. File System MCP
Поиск и чтение файлов в файловой системе.

### 5. Calendar MCP
Управление событиями и напоминаниями.

## Преимущества MCP

- Модульность: каждый сервер независим
- Расширяемость: легко добавлять новые серверы
- Стандартизация: единый протокол для всех серверов
- Безопасность: контроль доступа через конфигурацию

## Использование в God Agent

God Agent использует MCP для:
- Динамического подключения серверов из конфигурации
- Роутинга запросов к нужным инструментам
- Выполнения инструментов и получения результатов
- Предоставления списка доступных инструментов LLM

## Создание собственного MCP сервера

1. Создайте новый проект в `mcp-servers/`
2. Реализуйте интерфейс MCP сервера
3. Определите инструменты через `Tool` объекты
4. Добавьте обработчики для каждого инструмента
5. Зарегистрируйте сервер в конфигурации
